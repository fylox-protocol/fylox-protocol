# Fylox Protocol Architecture

Fylox is designed as a modular infrastructure layer that enables real-world payments within the Pi Network ecosystem.

The protocol architecture is composed of multiple layers that together enable real-world economic activity using Pi.

Fylox will evolve gradually. The first release focuses on a simple payment layer, while additional infrastructure layers will be introduced over time.

---

# Current Implementation (MVP)

The initial version of Fylox focuses on a single core capability:

**QR-based Pi payments.**

This MVP allows pioneers to:

• send Pi payments  
• pay merchants using QR codes  
• test real-world transaction flows  

The goal of this stage is to validate usability, reliability, and real-world adoption.

This payment layer is deployed as a lightweight web application compatible with Pi Browser.

---

# Long-Term Architecture

The full Fylox protocol architecture is designed around five layers.

These layers expand the protocol beyond simple payments and allow it to become real-world economic infrastructure.

The layers are introduced progressively as the ecosystem grows.

---

# 1. Oracle Layer (Future Layer)

The Oracle Layer connects real-world events with the digital ecosystem.

Pioneers can verify real-world actions and events through decentralized tasks.

Possible oracle use cases:

• delivery confirmations  
• location verification  
• real-world service validation  
• logistics events  

This layer bridges off-chain reality with digital trust.

---

# 2. Nexus Layer (Future Layer)

The Nexus Layer introduces a decentralized reputation system.

It allows verified Pi users to build trust and reputation within the network.

Potential components include:

• reputation scoring  
• activity history  
• identity validation through Pi KYC  
• trust indicators for services and transactions  

This layer helps create a reliable economic network.

---

# 3. Agora Layer (Future Layer)

The Agora Layer enables decentralized human governance.

Verified pioneers may participate in decision-making processes and validation tasks.

Possible use cases:

• dispute resolution  
• validation of AI-generated outputs  
• governance participation  
• service certification  

This layer introduces human consensus into the protocol.

---

# 4. Marketplace Layer (Future Layer)

The Marketplace Layer enables integrations with real-world services and applications.

Developers and partners can build services that connect to Fylox infrastructure.

Examples include:

• transportation services  
• utility payments  
• mobile top-ups  
• booking platforms  
• local commerce integrations  

This layer expands the real-world usability of Pi.

---

# 5. Fylox Payment Layer (Active Layer)

The Payment Layer is the first operational component of the protocol.

This is the interface used by pioneers and merchants to perform real-world payments.

Core features include:

• QR code payments  
• peer-to-peer transactions  
• merchant payment flows  
• integration with Pi wallets  

This layer forms the foundation of the Fylox ecosystem.

---

# Architectural Philosophy

Fylox follows a simple philosophy:

Start small.  
Validate real-world usage.  
Expand infrastructure gradually.

By beginning with a lightweight payment system and evolving into a broader economic infrastructure, Fylox aims to provide real utility for Pi.

---

Millions of pioneers already hold Pi.

The next step is simple:

Give them a place to spend it.

Fylox aims to become the infrastructure that enables everyday economic activity using Pi.
